Tsoding Base

Tsoding Base is a project of Tsoding subscribers to make a database of Tsoding's VODs

Contribution guideline

Tsoding Base contributer rules

If you want to openly contribute to Tsoding Base project you MUST follow these rules

  1. No docker bullshitery AT ALL
  2. Should use minimal amount of dependencies
  3. Interpreted languages are discouraged (except for JS frontend)
  4. No React
  5. Being subscribed to Tsoding's yt channel
  6. Use of languages that targed multiple platforms (ARM, ARM64, x86_64, RISC-V(?)) is encouraged
  7. Use open-source licenses
  8. Support each other developments and contribute if needed/requested
  9. Make dark theme option for frontend
  10. Making additions to minitags/timespamps of The Database if possible
  11. Idk yet

TsodingBase json file standard

    {
        "vods": [
            {
                "videotitle": "",
                "url": "",
                "languages": [],
                "project": "",
                "tags": [],
                "minitags": [],
                "timestamps": [{"tag":"", "time":"HH:mm:ss.hh", "end_time":"HH:mm:ss.hh"}],
                "date: ""
            },
            ...
        ]
    }
  • languages - the most important is the first
  • project - name or "none"
  • tags - also possible "nerdy" and "low level", should be in multiple form when appropriate "compilerS, debuggerS"
  • minitags - themes that were covered only in part of VOD but still are significant on some level, added minitag MUST be added to timestamps with "time" (and optional "end_time")
  • timestamps - "end_time" is optional, used for clips (Laur asked nicely)
  • date - date of upload, whatever yt-dlp tells you

Tagging rules

  1. if main topic is using a library, tags should contain "library"
  2. if non std library was used as a main topic of stream, tags must contain library's name
  3. "tools" should be added to tags if it was main topic of VOD
  4. tools name must be added to tags if it was main topic of VOD
  5. if stream introduced new library/language/tool/OS, tags must contain "new {library,language,tool,OS}" respectively
  6. if unsure about tag or if you think more tags need to be added add "???". I.e.: ["???"] or ["idk if this tag is needed???"]
  7. use "raylib speedrun" tag where appropriate

MiniTagging rules

  1. Any minitag added must have respectful tag in timestamps with at least "time" provided
